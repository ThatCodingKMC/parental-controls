# URL path patterns — blocked at all times (Python regex, one per line).
# These work on HTTPS sites too (via proxy).
# Lines starting with # are comments.

# YouTube: block Shorts.
#   1st line — direct navigation to youtube.com/shorts/<id>
#   2nd line — the Shorts-only internal API ("reel"); this is what kills
#   in-app swiping/tapping into Shorts. Regular videos use a different
#   endpoint (youtubei/v1/player + /next), so normal YouTube still works.
youtube\.com/shorts
youtubei/v1/reel

# ── Search engine porn backup patterns ───────────────────────────────────────
# Safe search is enforced at the proxy level (see proxy_addon.py), which is
# more robust. These patterns are a secondary layer catching obvious leakage.

# Block Google/Bing image search entirely (most common vector for explicit content)
(www\.)?google\.[a-z.]+/search\?.*tbm=isch
(www\.)?bing\.com/images/search
